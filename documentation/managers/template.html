Template Manager

The template manager is responsible for managing the assets (javascript and stylesheet files). The template manager is accessible through the script & stylesheet hooks.

Javascripts

The template manager allows you to add your own JavaScript files to SCADSY. You can add JavaScript files in the following scopes:

Scope Explanation Function
Global These scripts get loaded on every page of every controller of a every module. add_global_script($identifier, $url, $header = TRUE)
Module These scripts get loaded on every page of every controller of a certain module. add_module_script($identifier, $url, $module, $header = TRUE)
Controller These scripts get loaded on every page of a certain controller of a certain module. add_controller_script($identifier, $url, $module, $controller, $header = TRUE)
Method These scripts get loaded on a certain page of a certain controller of a certain module. add_method_script($identifier, $url, $module, $controller, $method, $header = TRUE)

As you can see, all the functions require an identifier for the javascript. This is meant to prevent script duplication. For example if "jquery" gets added 2 times the system will give out an error because jquery only needs to get included once.

add_global_script

With this function you can add a JavaScript file to the global scope. In the following example we add a new script to the global scope with the identifier "helloworld_script" and it is located at "modules/helloworld/assets/scripts/helloworld.js".

<?php
$template_manager->add_global_script('helloworld_script', 'modules/helloworld/assets/scripts/helloworld.js');
?>

add_module_script

With this function you can add a JavaScript file to the module scope. In the following example we add a new script to the module scope with the identifier "helloworld_script", located at "modules/helloworld/assets/scripts/helloworld.js" and for module "helloworld".

<?php
$template_manager->add_module_script('helloworld_script', 'modules/helloworld/assets/scripts/helloworld.js', "helloworld");
?>

add_controller_script

With this function you can add a javascript to the controller scope. In the following example we add a new script to the controller scope with the identifier "helloworld_script", located at "modules/helloworld/assets/scripts/helloworld.js", for module "helloworld" and for controller "hello".

<?php
$template_manager->add_controller_script('helloworld_script', 'modules/helloworld/assets/scripts/helloworld.js', "helloworld", "hello");
?>

add_method_script

With this function you can add a JavaScript file to the method scope. In the following example we add a new script to the method scope with the identifier "helloworld_script", located at "modules/helloworld/assets/scripts/helloworld.js", for module "helloworld", for controller "hello" and for the "index" method.

<?php
$template_manager->add_method_script('helloworld_script', 'modules/helloworld/assets/scripts/helloworld.js', "helloworld", "hello", "index");
?>

get_scripts

You can get the scripts in two states, the first state is an array and the second state is as html. By default, this function returns the header scripts as html. To get the footer scripts, change the first param to FALSE. To get the scripts as a list in stead of html set the second param to FALSE. So if you would want to get the list of the footer script you would call:

<?php
$template_manager->get_scripts(FALSE, FALSE);
?>

Stylesheets

The template manager also allows you to add stylesheets to SCADSY. The stylesheets, in comparison with the javascripts, don't have different scopes. Once you load a stylesheet, it will get loaded on every page. Please keep this in mind while naming your html elements. Stylesheets also have a identifier. Just as with the javascripts, this is to prevent duplicate stylesheets from loading.

add_stylesheet

To add a stylesheet file with identifier "helloworld_css" and located at "modules/helloworld/assets/style/helloworld.css" you can use the following code:

<?php
$template_manager->add_stylesheet("helloworld_css", "modules/helloworld/assets/style/helloworld.css");
?>

get_stylesheets

Just as with the scripts you can also get the array of stylesheets or the stylesheets as html. This is done by using the get_stylesheets function. By default this function returns the html of the stylesheets. If you would it to return the array you have to add the FALSE param. For example, the follow code will return the array of stylesheets:

<?php
$template_manager->get_stylesheets(FALSE);
?>
