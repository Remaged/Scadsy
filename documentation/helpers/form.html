Form Helper

The form helper of CodeIgniter is being extended in Scadsy.

set_value($field = '', $default = '')

Overwrites the set_value of CodeIgniter. It is used the same way. Instead of only repopulating the field with the matching POST-field when a form_validation is used, the field will now always be repopulated with the matching POST-field. Example:

<input type="text" name="quantity" value="<?php echo set_value('quantity', '0'); ?>" size="50" />

The above form will show "0" when loaded for the first time. After a submit, the form will show the submitted value (unless a different page is shown).

form_extra($identifier)

Selects a specific part of the extra_fields array that is stored in the form_manager (using the identifier) and outputs that array as a string. Example

<?php
echo form_extra("your_own_unique_form_identifier");
?>

This part will be filled by the form_manager. Everything that is stored in the form_manager using the identifier 'your_own_unique_form_identifier' is placed in this part. More on this can be found in the form_manager documentation.

form_any_input_type($data = '', $value = '', $extra = '', $type = 'text')

It's near the same as the form_input method of CodeIgniter's form_helper. This method has an additional $type parameter, allowing to use any type. This comes in handy when using specific html5 input-fields. Example

<?php
echo form_input('username', 'johndoe', '', 'email');

// Would produce:
<input type="email" name="username" value="johndoe" />
?>

form_date($data = '', $value = '', $extra = '')

This function is identical in all respects to the form_input() function above except that is sets it as a "date" type.

form_email($data = '', $value = '', $extra = '')

This function is identical in all respects to the form_input() function above except that is sets it as a "email" type.

form_search($data = '', $value = '', $extra = '')

This function is identical in all respects to the form_input() function above except that is sets it as a "search" type.

form_label_input($data = '', $human_name = NULL, $default_value = '', $extra = '')

Combines form_label and form_input and wraps them in a div. Example:

<?php echo form_label_input("username"); ?>

//Is the same as:
<div>
<?php echo form_label('Username','username'); form_input('username'); ?>
</div>

//Produces:
<div>
<label for="username">Username</label for="username">
<input type="text" name="username" />
</div>

form_label_{type}

Simular to form_password(), form_email(), form_date() and form_search() there are several variations on form_label_input(), which are: form_label_password, form_label_email(), form_label_date(), form_label_search()

form_label_any_type($data = '', $human_name = NULL, $default_value = '', $extra = '', $type = 'text'){

In case of needing a type that isn't supported (yet), you can use this method. With the $type parameter you can supply which type to use. Aside from the $type parameter it is identical to form_label_input()

form_label_textarea($data = '', $human_name = NULL, $default_value = '', $extra = '')

Simular to form_label_input(), except that it uses a textarea instead of an input.

form_label_dropdown($name = '', $options = '', $human_name = NULL, $default_value = '', $extra = '')

Combines form_label() and form_dropdown() and adds a div around them.

form_label_multiselect($name = '', $options = '', $human_name = NULL, $default_value = '', $extra = '')

Almost identical to form_label_dropdown, except that it adds a multiple attribute.
