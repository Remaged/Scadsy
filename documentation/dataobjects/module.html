Module

This object is the representation of the database "Module" table. To learn how to use this object please see the Datamapper explanation.

Fields

Below you will find information about the different fields the module has. You will also find the rules that are connected to each field. If you would want to save an module to the database make sure that every field abides the rules.

Property Information Type Rules
directory The directory the module resides in String
  • required
  • unique
  • 'max_size' => 100
  • 'min_size' => 1
name The name of the module String
  • required
  • 'max_size' => 100
  • 'min_size' => 1
uri The users first name String
  • 'max_size' => 200
description The modules description String
  • 'max_size' => 600
version The modules version String
  • 'max_size' => 20
author The author of the module String
  • 'max_size' => 100
author_uri The homepage of the author String
  • 'max_size' => 200
status The modules status String
  • required
  • 'valid_match' => array('enabled', 'disabled', 'not_installed')

Relations

Below you will find the foreign_key fields that the module has. For more information about the fields please look in the table above.

Field Relationship type
module_action one-to-many
module_permission one-to-many
