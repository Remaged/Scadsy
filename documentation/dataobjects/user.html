User

This object is the representation of the database "User" table. To learn how to use this object please see the Datamapper explanation.

Fields

Below you will find information about the different fields the user has. You will also find the rules that are connected to each field. If you would want to save an user to the database make sure that every field abides the rules.

Property Information Type Rules
id The unique number of the user. Integer
title The title of the person, for example: Mr String
  • required
  • 'valid_match' => array('Mr', 'Mrs', 'Ms')
first_name The users first name String
  • required
  • 'max_size' => 100
  • 'min_size' => 1
middle_name The users middle name String
  • 'max_size' => 100
last_name The users last name String
  • required
  • 'max_size' => 100
  • 'min_size' => 1
username The users username. String
  • required
  • unique
  • 'max_size' => 100
  • 'min_size' => 1
password The users encrypted password. String
  • required
email The users e-mail. String
  • required
  • 'max_size' => 100
  • 'min_size' => 1
phone_number The users phone number. String
  • required
  • 'max_size' => 40
  • 'min_size' => 1
date_of_birth The users date of birth. Date
  • required
  • valid_date
gender The users gender. String
  • required
  • 'valid_match' => array('male', 'female')
status The users status. String
  • required
  • 'valid_match' => array('enabled', 'disabled')
ethnicity_id ID of the ethnicity that is the user has. Int
  • 'max_size' => 11
language_id ID of the language that is the users main-language. Int
  • 'max_size' => 11

Relations

Below you will find the foreign_key fields that the user has. For more information about the fields please look in the table above.

Field Relationship type
ethnicity many-to-one
language many-to-one
grade many-to-one
group many-to-many
student many-to-one
parent many-to-one
teacher many-to-one
