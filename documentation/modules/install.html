Creating the install/uninstall script

So now that we have created our first module, we gotta give it some functionality. Sometimes you would like your module to have it's own database tables. Because they only have to be created once and it costs a lot of performance to check if a certain table exists every time we decided to let users create a install script. Having a install script is optional, if you don't create own your module will still work.

To create a install script follow the following steps:

  1. Go to the "module/helloworld" directory.
  2. In this directory create a new file called install.php
That's it! You know how created a install script. Everything that you put into this file will get executed when the module gets installed. But lets say we would like to create a new database table, we would need the main program object for that. So, let's explain how you can get it.

As we already told you earlier SCADSY was build on top of codeigniter. In codeigniter you can get the main object by calling <?php
$CI =& get_instance();
?>
So for simplicity, we left it that way. If you want to acces certain parts of the main program from inside your install script, simply get the main object and talk to it.
