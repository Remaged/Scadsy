Extra Features

Models

You should place all your models in the models directory. Models allow you to interact with the database through objects instead of queries. To find out more about using models inside SCADSY we would like to refer you to the datamapper page.

Helpers

You should place all your helpers in the helpers directory. All helper files should end with "_helper.php". Helpers are functions/methods that help you with certain repetitive tasks but are not part of a object and generally don't have any connection with the database.

In order to allow you to use object oriented programming while using hooks we implemented the following mechanic. If you place a file called callback_helper.php inside your helpers folder it will automatically be included during the loading of your module. Because of this it is possible to use callbacks inside your index.php to static functions declared inside your callback_helper.php.

Languages

If you want your module to support languages you should place the language files here. All language files should end with "_lang.php". Each language should have it's own folder, so it you want to support two languages your structure would look like the following:

  • language
    • english
      • helloworld_lang.php
    • dutch
      • helloworld_lang.php
Language files that have the same name as the module's main directory will be automatically included. In this case, if the module's main directory is called helloworld the files will be included. For the usage of languages we would like to refer you to the Language Helper.

Allow updating

SCADSY checks once every 24 hours if there is an update available for your module. In order to be able to check for updates your Module URI should be given inside the index.php file. Updating works the following way: (please make sure that, if you want to make use of updates, your module supports the following process)

  1. SCADSY calls MODULE_URI/?type=version to get the latest module version.
  2. SCADSY compares the two versions. If a newer version is available it will give notice to the user.
  3. If the user decided to install the update, SCADSY will call MODULE_URI/?type=file. It expects to get a .zip file back from this call.
  4. Once downloaded SCADSY will extract the .zip file inside the module's directory.
